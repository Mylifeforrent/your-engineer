# AI 辅助开发文档使用说明

> 这是整个开发文档包的入口文件。  
> 建议先阅读本文件，再按下列顺序使用其他文档。

## 1. 文档阅读顺序

### 第一次理解项目

1. `reference/01_PRD_V0.2.md`
   - 先理解产品定位、用户问题、MVP 边界和非目标。
2. `reference/02_系统架构设计_V0.1.md`
   - 再理解技术栈、模块边界、后台任务、LangGraph、数据库与部署方式。
3. `03_API_First开发规范.md`
   - 明确 API 契约、错误模型、版本控制和联调规则。
4. `04_MVP开发路线图与阶段拆分.md`
   - 了解四个阶段、MVP 切片、里程碑和前后依赖。
5. 当前阶段对应文档：
   - `phases/阶段一_基础架构与测试.md`
   - `phases/阶段二_前端开发.md`
   - `phases/阶段三_后端开发.md`
   - `phases/阶段四_接口联调与验收.md`
6. `api/openapi_mvp.yaml`
   - 前端、后端和测试共同使用的接口契约。
7. `05_MVP与MVP后开发边界.md`
   - 判断某个需求应进入 MVP 还是后续迭代。
8. `06_开发任务总清单.md`
   - 用于任务跟踪、状态更新和验收。

## 2. 使用 AI 开发时的文档优先级

当不同文档存在冲突时，按以下优先级执行：

1. 已评审并冻结的 `api/openapi_mvp.yaml`
2. 当前阶段任务文档及任务验收标准
3. 系统架构设计文档
4. PRD
5. 其他说明或历史讨论

产品需求发生变化时，不要直接让 AI 修改代码。先按以下顺序更新：

```text
PRD / 需求说明
    ↓
架构影响分析
    ↓
OpenAPI 合约
    ↓
前端类型和 Mock
    ↓
后端实现
    ↓
契约测试与联调
```

## 3. 每次给 AI 的最小上下文包

不要每次把全部文档交给 AI。针对一个具体任务，提供：

- 当前任务 ID 和目标
- 相关 PRD 小节
- 相关架构约束
- 对应 OpenAPI 路径与 Schema
- 验收标准
- 允许修改的目录
- 明确禁止修改的范围

推荐使用：

`templates/AI开发任务输入模板.md`

## 4. 推荐开发顺序

### 阶段一

先建立前后端骨架、数据库、测试体系、统一错误模型和 OpenAPI 契约。

阶段一结束时必须达到：

- 前端可以启动
- 后端可以启动
- PostgreSQL 可连接
- 健康检查通过
- OpenAPI 合约通过评审
- 前端能使用 Mock 数据显示一个最小页面
- 后端能返回一个符合契约的最小接口

### 阶段二与阶段三

API 合约冻结后，前端与后端可以并行开发：

- 前端依据 OpenAPI 类型和 Mock 数据完成页面
- 后端依据同一 OpenAPI 完成业务实现
- 任何接口变更必须先修改并重新评审 OpenAPI

虽然文档按“前端阶段、后端阶段”分别组织，但实际执行可并行。

### 阶段四

切换真实 API，执行：

- 契约校验
- 字段和错误处理联调
- 完整业务链路 E2E
- UAT
- MVP 发布验收

## 5. MVP 开发建议

先完成两个纵向切片：

1. **Release MVP**
   - 创建 Release Case
   - 收集 Jira / Jenkins 基础数据
   - Checklist
   - AI 草稿
   - Confluence 发布
   - Change Request 草稿与创建
2. **Alert MVP**
   - 接收 xMatters 告警
   - 系统映射
   - Elasticsearch / Prometheus 查询
   - 最近变更
   - AI 摘要与最多三个假设
   - 用户反馈

不要先做大量配置页面或抽象框架，再做业务流程。每个切片应尽快做到端到端可运行。

## 6. AI 编码工作方式

推荐让 AI 按以下粒度工作：

- 一次完成一个任务 ID
- 先输出实施计划和受影响文件
- 再修改代码
- 修改后运行测试
- 最后输出：
  - 已完成内容
  - 测试结果
  - 未完成或风险
  - API / 数据库是否发生变化

避免让 AI 一次生成整个系统。大型一次性生成难以保证契约、迁移、测试和错误处理一致。

## 7. Definition of Done

一个开发任务只有同时满足以下条件才算完成：

- 代码完成
- 类型检查通过
- 单元测试通过
- 与 OpenAPI 一致
- 错误路径已处理
- 日志不泄露 Secret
- 文档或注释已更新
- 验收标准逐条验证
